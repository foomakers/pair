Tutorials are learning-oriented, guided walkthroughs that you complete in a single session. Each one takes you from start to finish with a working result.

Different from [Getting Started](/docs/getting-started) quickstarts (5–10 minutes), tutorials go deeper (30–60 minutes) and cover the full workflow end-to-end.

## Pick your tutorial

<Cards>
  <Card title="Your First Project" href="/docs/tutorials/first-project">
    **Solo developer** · ~30 min — Install pair, bootstrap a project, create a PRD, generate your first initiative/epic/story, and run your first skill.
  </Card>
  <Card title="Team Setup" href="/docs/tutorials/team-setup">
    **Team lead** · ~45 min — Install a shared Knowledge Base, customize guidelines, configure PM tool integration, and establish the team's way-of-working.
  </Card>
  <Card title="Enterprise Adoption" href="/docs/tutorials/enterprise-adoption">
    **Enterprise architect** · ~60 min — Create an org-level KB, structure it for multiple teams, distribute via npm, set governance policies, and onboard the first team.
  </Card>
</Cards>
